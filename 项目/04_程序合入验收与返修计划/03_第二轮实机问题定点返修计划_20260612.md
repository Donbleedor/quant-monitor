## 2. 实机问题证据

### 2.1 菜单构图过度集中在上半屏

`360x800` 实测：

| 元素 | 当前范围 | 问题 |
| --- | --- | --- |
| Logo wrapper | `y=28..171` | 过高 |
| Play | `y=181..255` | 过高 |
| Levels / Daily | `y=267..377` | 过高 |

当前下半屏出现约 400px 的无目的空区，与批准效果图不符。

### 2.2 弹窗被物理宽度锁死

`720x1600` 实测：

- app：`720x1600`
- modal：`460x548`
- modal 只占 app 宽度约 `64%`

原因：`.modal { max-width: 460px; }` 阻断 `--ui-scale`。

### 2.3 Levels 实际只有三列

`720x1600` 实测：

- `.modal-content` 宽度：`294px`
- `.level-node`：`88px`
- gap：`16px`
- 实际只能容纳 3 列

CSS 虽写 `repeat(4, ...)`，但 `min-width` 与 gap 使四列约束无法成立。

### 2.4 Settings 按钮溢出

`720x1600` 实测：

- modal：`y=526..1074`
- Reset：`y=1007..1092`

Reset 底部超出面板，且文字换成两行后压住底边。

### 2.5 HUD 信息没有落在正确槽位

实机截图中：

- `Lv.` 文案挤在左侧圆槽。
- 中央长板基本空置。
- timer 图标和时间一起占用右圆槽。

目标是左圆槽只放关卡图标，中央长板显示 `LV. N` 和 `M:SS`，右圆槽只放 timer 图标。

### 2.6 工具图标语义错误

当前资源内容：

| 文件 | 实际图形 |
| --- | --- |
| `icon_hint.png` | 暂停符号 |
| `icon_undo.png` | 灯泡 |
| `icon_shuffle.png` | 回退箭头 |

因此当前画面表现为：

- Hint = 暂停
- Undo = 灯泡
- Mix = 回退

这会直接误导操作。

### 2.7 Settings 伪功能

- SFX / Music 只有保存状态，没有音频系统。
- Large Text / Contrast 虽给 `#app` 加 class，但 CSS 没有消费这些 class。
- Reset 仍使用浏览器原生 `confirm()`。
