# Mahjong V3 UI Reskin Correction Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变麻将消除核心玩法的前提下，修复资源残留、组件误用、牌面状态、交互语义和 9:20 内部适配，使实装接近批准效果图并恢复清晰可玩的牌阵。

**Architecture:** 保留 `engine.js` 和 34 个逻辑 typeId。以 `360x800` 为逻辑设计坐标，通过 app 容器缩放变量驱动 HUD、牌阵、底栏和弹窗；资源层、状态层和交互层分别返修，最后以同尺寸截图对比验收。

**Tech Stack:** 原生 HTML、CSS、JavaScript、PNG；不新增游戏框架。

## 2. 统一适配基线

逻辑画布：

- `360x800`
- 生产参考：`1080x2400`
- 低端机参考：`720x1600`

逻辑区域：

| 区域 | 建议逻辑尺寸 |
| --- | --- |
| Top HUD | `0-58` |
| 呼吸间隔 | `58-70` |
| Board | `70-676` |
| Bottom Tray | `676-800` |

推荐缩放方式：

```js
const scale = app.clientWidth / 360;
app.style.setProperty('--ui-scale', scale);
```

```css
#app {
  container-type: size;
}

.game-screen {
  --s: var(--ui-scale, 1);
}
```

如果不使用 JS 变量，也必须保证所有内部尺寸由 app 容器计算，不能继续混用浏览器 `vw` 和固定像素。

## 5. Task 3：修复 9:20 内部等比缩放

**Files:**

- Modify: `style.css:21-24, 46-57`
- Modify: `ui.js:129-159`

- [ ] 保留当前 `#app` 的 9:20 contain 逻辑。
- [ ] 增加基于 app 宽度的统一缩放变量。
- [ ] 将 `zOffsetPx = 4` 改为 `4 * scale`。
- [ ] 删除牌宽绝对上限 `64` 和下限 `36`。
- [ ] 使用设计上限 `56 * scale`，最小可接受值只用于 QA 报警，不用于强行撑大。
- [ ] 把固定的 modal 星星、padding、gap 和 toast 尺寸迁移到同一缩放体系。
- [ ] 字体基于 app/container 缩放，不基于浏览器 `vw`。

推荐牌尺寸公式：

```js
const scale = appRect.width / 360;
const zOffset = 4 * scale;
const widthFit = (availW - maxZ * zOffset) / gridW;
const heightFit = (availH - maxZ * zOffset) / gridH;
const tileW = Math.min(widthFit, heightFit * 168 / 201, 56 * scale);
const tileH = tileW * 201 / 168;
```

通过标准：

- app 宽度翻倍时，牌、层高、图标和文字同步接近翻倍。
- 桌面大窗口不再出现“外框很大、牌阵仍是 64px”的情况。
