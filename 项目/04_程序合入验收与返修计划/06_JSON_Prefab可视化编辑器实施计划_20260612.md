# Mahjong V3 JSON Prefab Visual Editor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不修改玩法引擎的前提下，为 Mahjong V3 增加通过 `?editor=1` 启用的 JSON Prefab 可视化布局编辑器，并保证普通游戏只读取默认布局配置。

**Architecture:** 使用现有 CSS 作为基础 Prefab，JSON 只保存现有组件的布局覆盖。`layout-core.js` 处理纯数据，`layout-runtime.js` 将 360x800 逻辑坐标应用到真实 DOM，`layout-editor.js` 在桌面模式提供组件树、RectTransform、属性编辑、历史和 JSON 导入导出。

**Tech Stack:** 原生 HTML、CSS、JavaScript、JSON、Node 内置 `assert`；不引入框架、构建工具或第三方依赖。

## 19. 完成定义

必须同时满足：

- 普通游戏无视觉和行为回退。
- 普通模式不加载编辑器资源。
- 编辑器只能由 `?editor=1` 启用。
- 只编辑现有组件。
- 360x800 是唯一保存坐标。
- 720/1080 自动等比。
- 草稿只影响编辑模式。
- JSON 导入导出可往返。
- 页面重新渲染后布局仍应用。
- Board 不允许逐牌拖动。
- 单元测试通过。
- 浏览器报告为 `EDITOR_ACCEPTED`。

任何一项不满足，都不得宣称完成。
